# 实现一个类似 typeless 的语音输入工具

## 需求介绍

我准备做一个Mac上语音输入的客户端产品，简要功能如下：
- 用户按住 Fn 键的时候，桌面底部中心位置出现一个语音聆听条，然后用户可以说话
- 再次按起 Fn 的时候，结束语音输入
- 然后使用本地语音模型将语音转换为文字
- 再使用外部大模型如GPT将文字进行优化，比如进行结构化、去除口吃、语气助词等，保留最关键的文本信息。
- 最后将文本填入到用户当前系统聚焦的输入框中

除此之外，你可以访问 https://www.typeless.com/ 来了解 typeless 的功能和设计，以便参考其功能更好地实现类似的语音输入工具。

但注意我需要语音识别是本地模型，不能依赖云端服务，只有最后的文本优化才可以使用外部大模型。

最后，登录注册支付功能不需要实现，但有一个用户填入API Key的界面，用户可以在这个界面填入自己的OpenAI API Key & Base url，以便后续使用GPT进行文本优化。

## 技术方案

我个人熟悉的技术栈的前端技术栈，如Next.js、Electron、Tarui、Nest.js等，你可以使用 monorepo 的方式来组织前后端代码，或者你也可以选择其他你熟悉的技术栈来实现这个项目，但最好你先在github上调研，有没有一些现成的优秀开源模版进行参考（> 100 stars），也可以参考多个开源模版进行初始代码的组织

## 测试实现

客户端的交互暂时不需要测试，但是API实现需要完整的测试，只有测试通过才算完成，具体的测试用例你需要先编写，再进行代码实现，测试用例需要覆盖到所有的边界情况和异常情况，确保代码的健壮性和可靠性。
